# Chapter 0. Welcome to fluo: The Standard-First Framework

**fluo**에 오신 것을 환영합니다. fluo는 웹 개발의 향후 10년을 위해 바닥부터 설계된 현대적인 TypeScript 백엔드 프레임워크입니다. 기존 프레임워크의 "마법"과 레거시 부채 없이 확장 가능하고 고성능이며 미래 지향적인 서버측 애플리케이션을 구축하는 방법을 찾고 있다면 제대로 찾아오셨습니다.

이 책은 여러분을 단순한 호기심을 가진 개발자에서 능숙한 fluo 아키텍트로 성장시키기 위해 설계되었습니다. 우리는 단순히 문법만 살펴보는 것이 아니라, **FluoBlog**라는 실제 세계에서 즉시 사용 가능한 프로덕션급 애플리케이션을 함께 구축할 것입니다. 이 과정에서 왜 표준이 중요한지, 유지보수성을 위해 명시적 설계(Explicit Design)를 어떻게 활용하는지, 그리고 Node.js부터 Edge까지 모든 런타임에 코드를 배포하는 방법을 배우게 될 것입니다.

## What is fluo?

코드로 들어가기 전에 fluo가 무엇인지, 무엇이 다른지 정의해 보겠습니다. 오늘날 대부분의 TypeScript 프레임워크는 수년 전에 제안되었지만 공식 JavaScript 언어의 일부가 되지 못한 실험적인 기능들에 의존합니다. `tsconfig.json` 파일에서 `experimentalDecorators`나 `emitDecoratorMetadata` 같은 설정을 본 적이 있을 것입니다. 이러한 기능들은 당시에는 혁신적이었지만, 이제는 상당한 아키텍처적 무게를 지니게 되었고 표준과 항상 일치하지 않는 특정 컴파일러 동작을 요구합니다.

fluo는 **표준 우선(Standard-First)**을 지향함으로써 이 고리를 끊습니다.

fluo는 완전히 **TC39 Stage 3 Decorator** 사양을 기반으로 구축되었습니다. 이것은 단순히 기술적인 세부 사항이 아니라, 메타데이터와 동작을 코드에 연결하는 방식의 근본적인 전환입니다. 컴파일러 트릭이 아니라 실제 JavaScript 런타임의 일부가 될 공식 기능을 사용함으로써, fluo는 이전에는 불가능했던 수준의 안정성과 성능을 달성합니다.

그 결과로 탄생한 프레임워크는 다음과 같은 특징을 가집니다.

- **Lean**: `reflect-metadata`와 같은 무거운 reflection 라이브러리나 숨겨진 메타데이터 부풀리기가 없습니다. 번들 크기가 작게 유지됩니다.
- **Fast**: AWS Lambda나 Vercel 같은 서버리스 환경에서 매우 중요한 "Cold Start" 시간이 단축되며, 메모리 사용량도 현저히 낮습니다.
- **Explicit**: 프로젝트 전체를 마법처럼 스캐닝하는 일이 없습니다. 모듈 정의만 봐도 의존성이 어떻게 연결되어 있는지 정확히 확인할 수 있습니다.
- **Portable**: 동일한 코드가 Node.js, Bun, Deno, 그리고 Cloudflare Workers에서 실행됩니다. fluo는 Platform Adapter Contract를 사용하여 이러한 런타임 간의 차이를 처리하므로, 여러분의 비즈니스 로직은 플랫폼에 종속되지 않고 순수하게 유지될 수 있습니다.

## Why This Book?

fluo에 대한 문서는 이미 많이 존재하지만, 공식 문서는 종종 특정 기능이 "어떻게" 작동하는지에 집중합니다. `@Get()` 데코레이터의 API 레퍼런스는 금방 찾을 수 있겠지만, 그것을 언제 사용해야 하는지, 데이터를 처리하기 위해 서비스를 어떻게 구성해야 하는지, 그리고 그 로직을 어떻게 테스트해야 하는지를 배우는 것이 진짜 도전입니다.

이 책은 **"왜(Why)"**와 **"흐름(Flow)"**에 집중합니다.

새로운 프레임워크를 배우는 것이 부담스러울 수 있다는 점을 잘 알고 있습니다. 특히 Express나 NestJS에서 배웠던 패턴들에 익숙하다면 fluo의 명시성이 생소할 수도 있습니다. 그래서 이 책은 누적된 경로를 따릅니다. 모든 기능을 한꺼번에 쏟아붓지 않고, 절대적인 기초—환경 설정과 핵심 철학 이해—부터 시작하여 복잡성의 층을 점진적으로 쌓아 나갑니다.

이 책을 일종의 "가이드가 있는 도제 교육"이라고 생각하세요. 이 시리즈를 마칠 때쯤이면 여러분은 단순히 fluo를 사용하는 법을 아는 것이 아니라, 백엔드 시스템을 견고하고 확장 가능하며, 무엇보다 수년간의 개발 과정에서도 유지보수 가능하게 만드는 아키텍처 패턴을 이해하게 될 것입니다.

## The FluoBlog Project

이 책의 핵심은 **FluoBlog**입니다. 단편적이고 "할 일 목록(To-do list)" 스타일의 예제 대신, 우리는 21개 챕터에 걸쳐 완전한 프로덕션급 블로그 엔진을 구축할 것입니다. 이것은 단순한 튜토리얼 프로젝트가 아니라, 실제 전문 기술 기업에서 구축할 법한 시스템의 축소판입니다.

우리는 다음과 같은 것들을 구현할 것입니다.

1. **A Modular Architecture**: 코드를 "거대한 진흙탕"으로 만들지 않고 논리적이고 분리된 단위로 구성하는 방법.
2. **RESTful APIs**: 복잡한 HTTP 요청, 상태 코드, 헤더를 정밀하게 처리하기.
3. **Database Integration**: 현대적인 ORM인 Prisma를 사용하여 완벽한 타입 안정성으로 영구 데이터 관리하기.
4. **JWT Authentication**: JSON Web Token과 전략 기반 권한 부여를 사용하여 API 보안 강화하기.
5. **Caching**: Redis를 사용하여 높은 트래픽 시나리오를 처리할 수 있도록 성능 끌어올리기.
6. **Observability**: 헬스 체크, 구조화된 로깅, Prometheus 메트릭을 추가하여 프로덕션에서 실제로 무슨 일이 일어나는지 파악하기.

각 챕터는 FluoBlog에 새롭고 구체적인 기능을 추가합니다. 이러한 방식은 실제 개발 라이프사이클을 반영하며, 단일 파일에서 시작한 코드베이스가 어떻게 정교한 시스템으로 진화하는지 보여줍니다.

## Prerequisites

이 책을 최대한 활용하려면 다음이 필요합니다.

- **기초적인 JavaScript/TypeScript 지식**: 전문가일 필요는 없지만, 클래스, `async/await`, 기본적인 타입 선언에 익숙해야 합니다.
- **Node.js 설치**: fluo는 많은 런타임을 지원하지만, 우리는 Node.js(18 버전 이상)와 `pnpm`을 기본 개발 환경으로 사용합니다.
- **터미널과 코드 에디터**: 최고의 개발 경험을 위해 TypeScript 확장이 설치된 VS Code를 권장합니다.

NestJS, Express 또는 다른 백엔드 프레임워크에 대한 사전 경험은 **필요하지 않습니다**. 오히려 다른 프레임워크를 경험해 보셨다면 fluo의 명시성이 신선하게 다가올 수도 있습니다. 우리는 웹이 어떻게 작동하는지에 대한 기초 지식 외에는 백엔드 지식이 전혀 없다고 가정하고 모든 개념을 바닥부터 설명합니다.

### The Philosophy of "No Magic"

fluo를 사용하면서 가장 먼저 느끼게 될 특징 중 하나는 "마법"이 없다는 점입니다. 많은 인기 프레임워크에서는 여러분의 명시적인 지시 없이도 배후에서 많은 일이 일어납니다. 이것이 처음에는 강력하게 느껴질 수 있지만, 문제가 발생했을 때는 큰 혼란을 야기합니다.

fluo에서는 여러분이 애플리케이션의 아키텍처를 직접 제어해야 한다고 믿습니다. 서비스에 데이터베이스가 필요하다면 fluo에 이를 제공하라고 명시적으로 지시합니다. 컨트롤러가 특정 경로를 처리해야 한다면 그 경로를 명시적으로 정의합니다. 이러한 명시성은 코드를 읽기 쉽게 만들고, 테스트하기 쉽게 하며, 프로젝트가 커져도 유지보수를 훨씬 수월하게 만듭니다.

마법을 제거함으로써 우리는 여러분에게 코드를 논리적으로 추론할 수 있는 능력을 되돌려 줍니다. 의존성이 왜 주입되지 않았는지, 왜 경로가 작동하지 않는지 추측할 필요가 없습니다. 답은 언제나 여러분의 소스 코드 안에 명확하고 감사 가능한 형태로 존재할 것입니다.

### A Framework for Every Environment

현대의 웹은 더 이상 전통적인 서버에만 국한되지 않습니다. 이제 우리는 서버리스 함수, 엣지 런타임, 심지어 IoT 기기와 같은 특수한 환경에도 코드를 배포합니다. fluo는 이러한 다양성을 염두에 두고 구축되었습니다.

우리의 "런타임 중립적(Runtime-Neutral)" 접근 방식은 애플리케이션의 핵심—비즈니스 로직, 서비스, 컨트롤러—이 어디에서 실행되는지 신경 쓰지 않는다는 것을 의미합니다. 고성능 Node.js 클러스터에 배포하든, 가벼운 Cloudflare Worker에 배포하든, 여러분의 fluo 코드는 정확히 동일하게 유지됩니다.

이러한 이식성은 우리의 Platform Adapter Contract를 통해 달성됩니다. 우리는 다양한 런타임 간의 차이를 추상화하는 어려운 작업을 대신 처리했으므로, 여러분은 플랫폼별 API와 싸우는 대신 기능을 구현하는 데 집중할 수 있습니다.

### The Value of Standard-First

"표준 우선" 프레임워크를 선택하는 것은 여러분의 개발 커리어를 위한 전략적인 결정입니다. fluo를 배울 때, 여러분은 단순히 독자적인 도구를 배우는 것이 아니라 미래의 공식 JavaScript 표준을 배우는 것입니다.

TC39 Stage 3 데코레이터 사양은 우리 프레임워크의 근간입니다. fluo를 마스터함으로써 여러분은 향후 수년간 JavaScript 개발을 정의할 네이티브 언어 기능에 대한 깊은 전문성을 얻게 됩니다. 이 지식은 다른 곳에서도 유용하게 쓰일 수 있으며 미래 지향적입니다.

우리는 독자적인 문법을 발명하는 프레임워크들이 가지는 "락인(lock-in)" 효과를 피합니다. fluo와 함께라면 여러분은 언제나 언어의 의도에 가깝게 코딩하게 됩니다. 표준과의 이러한 정렬은 생태계가 어떻게 변하더라도 여러분의 기술이 계속 유효하도록 보장해 줄 것입니다.

### Your Journey Starts Here

능숙한 백엔드 개발자가 되는 것은 단거리 경주가 아니라 마라톤입니다. fluo는 이 여정에서 견고한 기초와 명확한 길을 제공하는 동반자가 되도록 설계되었습니다.

이어지는 챕터들에서 여러분은 바닥부터 무언가를 구축하는 즐거움을 경험하게 될 것입니다. 여러분의 코드가 살아 움직이는 것을 보는 만족감, 그리고 애플리케이션이 어떻게 작동하는지 정확히 이해하고 있다는 자신감을 느끼게 될 것입니다.

여러분을 fluo 커뮤니티의 일원으로 맞이하게 되어 매우 기쁩니다. 이제 FluoBlog 구축을 시작해 봅시다. 그 과정에서 여러분은 유능한 fluo 아키트로 거듭나게 될 것입니다.

## How to Read This Book

이 책은 여러분의 숙련도를 한 단계씩 높여줄 5개의 논리적인 파트로 구성되어 있습니다.

### Part 0. Getting Started
fluo 설계 뒤에 숨겨진 "왜"를 다루고, CLI를 설정하며, 핵심 빌딩 블록인 Module, Provider, Controller를 소개합니다. 또한 fluo의 "비법 소스"인 데코레이터의 신비를 벗겨내고, 다른 곳에서 보았을 레거시 데코레이터와 어떻게 다른지 설명합니다.

### Part 1. Building the HTTP API
여기서는 애플리케이션의 "얼굴"을 만듭니다. 라우팅, 데이터 전송 객체(DTO)를 통한 사용자 입력 처리, 데이터 자동 검증 및 일관된 응답 반환에 대해 배웁니다. 또한 OpenAPI(Swagger)를 사용하여 API 문서를 자동으로 생성하고 호스팅하는 방법도 다룹니다.

### Part 2. Configuration and Data
데이터베이스 없는 백엔드는 불완전합니다. 개발 및 프로덕션 등 각 단계에 맞는 환경 기반 설정을 구축하고 Prisma를 사용하여 PostgreSQL 데이터베이스와 통신할 것입니다. Repository 패턴과 데이터베이스 트랜잭션을 안전하게 처리하는 방법을 배우게 됩니다.

### Part 3. Authentication and Security
보안은 fluo에서 나중에 덧붙이는 것이 아닙니다. 강력한 JWT 인증을 구현하고, Passport를 사용하여 유연한 보안 전략을 세우는 법을 배우며, Rate Limiting을 통해 무차별 대입 공격과 같은 위협으로부터 API를 보호합니다.

### Part 4. Caching and Operations
마지막으로 FluoBlog를 실제 세계에 내놓을 준비를 합니다. Redis 캐싱 레이어를 추가하여 빈번한 요청을 빛의 속도로 처리하고, 로드 밸런서를 위한 표준화된 헬스 체크를 구현하며, Prometheus 메트릭을 설정하여 애플리케이션 상태를 실시간으로 추적합니다.

### Part 5. Testing and Completion
비즈니스 로직에 대한 단위 테스트와 API 엔드포인트에 대한 통합 테스트를 작성하는 것으로 마무리합니다. 규모가 커짐에 따라 블로그에 버그가 없도록 유지하는 것이 퍼즐의 마지막 조각이며, 이어서 프로덕션 배포 체크리스트를 살펴봅니다.

## Using the Code Examples

모든 챕터에는 세심하게 선별된 코드 스니펫이 포함되어 있습니다. 이를 최대한 활용하기 위해 몇 가지를 권장합니다.

- **직접 타이핑하세요**: 구식처럼 들릴 수도 있지만, 단순히 복사해서 붙여넣지 마세요. 코드를 직접 치는 것은 "근육 기억"을 돕고 문법과 패턴의 작은 세부 사항들을 알아차리게 해줍니다.
- **망가뜨려 보고 실험하세요**: 챕터에서 `@Get()` 경로를 보여주면, `@Post()`로 바꿔 보거나 커스텀 헤더를 추가해 보세요. 필수 Provider를 누락했을 때 어떤 일이 일어나는지 확인해 보세요. fluo의 에러 메시지는 도움을 주도록 설계되었으며, 이를 읽는 법을 배우는 것은 필수적인 기술입니다.
- **공식 저장소를 확인하세요**: 막히는 부분이 있다면 fluo 공식 저장소의 `examples/` 디렉터리에 프로젝트 단계별 완성된 코드가 있습니다. 자신의 구현과 비교하여 어디가 달라졌는지 확인해 보세요.

## Community and Support

fluo 커뮤니티는 표준, 성능, 그리고 깨끗한 코드를 중시하는 개발자들의 모임입니다. 이 여정에서 여러분은 혼자가 아닙니다.

- **GitHub Discussions**: 일반적인 질문, 아키텍처 조언을 구하거나 여러분이 구축한 것을 뽐내기에 가장 좋은 장소입니다.
- **Issue Tracker**: 프레임워크의 버그나 책의 예제에서 오류를 발견하면 알려주세요! 우리는 문서의 버그도 코드의 버그만큼 중요하게 생각합니다.
- **Discord**: 다른 개발자 및 핵심 유지관리자들과 실시간으로 소통할 수 있는 채널입니다. 어려운 개념에 대해 빠르게 확인하고 싶을 때 좋습니다.

## Orientation: The fluo Package Ecosystem

입문자들이 놀라는 것 중 하나는 fluo가 하나의 거대한 "블랙박스"가 아니라는 점입니다. 대신, fluo는 39개 이상의 전문화되고 상호 운용 가능한 패키지들의 모음입니다. 이러한 모듈성은 설계된 것입니다—실제로 사용하는 코드만 포함하도록 하여 애플리케이션을 가볍게 유지합니다. 이 입문 시리즈에서 우리는 주로 "핵심 4인방(Core Four)"에 집중합니다.

- `@fluojs/core`: 모듈 시스템과 의존성 주입을 제공하는 기반.
- `@fluojs/http`: 웹 서버 구축 및 HTTP 트래픽 처리와 관련된 모든 것.
- `@fluojs/cli`: 새 프로젝트 스캐폴딩 및 컴포넌트 생성을 위한 명령줄 도구.
- `@fluojs/di`: 클래스들을 명시적으로 연결하는 강력한 엔진.

이 책을 마칠 때쯤이면 여러분은 이 조각들이 어떻게 조화를 이루는지, 그리고 프로젝트에 필요할 때만 추가 패키지(예: `@fluojs/prisma` 또는 `@fluojs/redis`)를 어떻게 가져오는지 이해하게 될 것입니다.

## Setting Expectations

이 책은 여러분을 fluo 전문가로 성장시키기 위해 설계된 포괄적인 3부작 시리즈 중 첫 번째 권입니다.

- **Book 1 (Beginner)**: 기능 구축과 표준 개발자 워크플로 마스터하기에 집중합니다. 일상적인 fluo 개발의 "방법"을 배우게 됩니다.
- **Book 2 (Intermediate)**: 마이크로서비스, 커스텀 데코레이터, 고급 DI 스코프(Request/Transient), 복잡한 이벤트 기반 아키텍처와 같은 더 복잡한 주제를 깊게 다룹니다.
- **Book 3 (Advanced)**: 프레임워크 내부 원리를 파헤쳐 커스텀 플랫폼 어댑터를 직접 구축하고, 극한의 대규모 성능 튜닝을 하는 방법을 다룹니다.

이 첫 번째 권의 목표는 **자신감**입니다. 21챕터를 마칠 때쯤이면 여러분은 바닥부터 새로운 백엔드 프로젝트를 시작하고 프로덕션 환경까지 이끌 수 있는 충분한 준비를 갖추게 될 것입니다.

### A Note on the "Standard-First" Approach
우리가 "표준 우선"이라고 말할 때, 그것은 개발자로서의 여러분의 커리어에 대한 약속이기도 합니다. fluo를 배움으로써 여러분은 공식 JavaScript Decorator API를 배우는 것입니다. 나중에 다른 도구나 다른 언어로 옮겨가더라도, 여기서 배우는 패턴들—의존성 주입, 모듈화, 명시적 설정—은 보편적으로 적용됩니다.

많은 개발자가 독자적인 DSL(Domain Specific Language)을 사용하는 프레임워크에 "갇혀 있다"고 느낍니다. fluo는 그 반대입니다. fluo는 여러분이 이미 알고 있는 언어의 확장입니다.

### Why Explicitness Matters
웹 개발 초기에는 "마법"이 기능으로 여겨졌습니다. 개발자가 무엇을 원하는지 추측할 수 있는 프레임워크가 인기를 끌었습니다. 하지만 애플리케이션이 거대한 마이크로서비스로 성장하면서 그 마법은 악몽이 되었습니다. 마법은 디버깅을 불가능하게 만들었고 리팩토링을 도박으로 만들었습니다.

fluo는 다른 길을 선택합니다. 우리는 **명시적인 것이 암시적인 것보다 낫다**고 믿습니다. fluo 컨트롤러를 보면 데이터가 어디서 오는지 정확히 알 수 있습니다. 모듈을 보면 그 모듈이 무엇을 제공하는지 정확히 알 수 있습니다. 처음에는 몇 줄의 코드가 더 필요할 수 있지만, 나중에 수백 시간의 디버깅 시간을 절약해 줍니다.
